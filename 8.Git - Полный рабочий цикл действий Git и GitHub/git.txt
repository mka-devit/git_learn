как обычно проходит решение тасков
дается доступ к частной репозитории, ты ее клонируешь
```
git clone git@github.com:mka-devit/docker.git
```
далее создаешь новую ветку чтобы не изменить основную ветку
часто дают имя ветке `имя кто редактирует и номер таски с jira`
```
git chekout -b max_tash007
```
далее редактируешь файл изменив в нем все что нужно
```
git add .
```
далее комитешь
commit нужно давать понятный что именно ты сделал указать 
```
git commit -m "add_file5 edit_index"
```
далее свою ветку отправляешь на удаленный репозиторий
с помошью `git push`
```
git push origin
```
возможно выдаст ошибку так как твоей ветки может не быть 
и набираешь команду чтобы создать ветку в удаленном репозитории
```
git push --set-upstream origin <branch_name>
```

#удаление не нужных веток с локальной и удаленного репозитория
для начало нужно переключиться на основную ветку
```
git checkout main
```
далее удаляем локальную не нужную ветку
```
git branch -d maxtask
```
теперь можно удалить на удаленном репозитории
```
git push origin --delete maxtask
```
