#Git - Работа с Local Repository
чтобы git начал следить за обновлениями
просто нужно создать базу данных git
```
git init .
```
`.` - точка означает (сделать базу данных в той директории в которой я нахожусь)
либо можно прописать полный путь
_____________________________________
чтобы провеерить состояния файлов в git
```
git status
```
_____________________________________
#Что именно делает `git status`:
- Показывает **изменённые файлы**, которые **ещё не добавлены в коммит**.
- Показывает **файлы, добавленные в индекс** (через `git add`), и готовые к коммиту.
- Показывает **неотслеживаемые файлы** (т.е. новые файлы, о которых Git пока "не знает").
- Указывает, есть ли **различия между локальной и удалённой веткой** (если настроен удалённый репозиторий).
- Показывает, в какой **ветке** вы сейчас находитесь.
_____________________________________
ветка master - это золотая копия самая оснавная главная ветвь
чтобы дальше с ней работать
```
git add .
```
если хотим добавить один файл пишим `git add file1.txt`
если все `git add .` или `git add *`
после чего файлы будет на staging status
В Git **staging status** относится к файлам, которые находятся 
в **области индексирования** — **staging area** (она же **index**). 
Это промежуточная зона, где ты подготавливаешь изменения к коммиту.
_____________________________________
чтобы удалить с него пишим команду:
```
git rm --cached <file>
```
_____________________________________
так чтобы теперь снять копию файлов
пишем:
```
git commit -m "My First Commit"
```
