1) Файл который мы только что создали на стороне GIT он будет в stage (этап) - Untracked
2) Когда мы запускаем `git add` - файл будет находится на stage - Staged
3) Когда мы набираем `git commit -m "massege"` файл переходит на этап - Copy in Local Repository
4) И четвертое `git push` он переводит наш local repository в -  Copy on Remote Repository
#История изменения
просмотр истории всех изменений
```
git log
```
чтобы посмотреть самый последний лог изменений
```
git log -1
```
если `git log -2` : покажет 2 последних commita
_____________________________________
посмотреть что было изменено  в файлах
`-p` 
```
git log -1 -p
```

#восстановление файлов

ну например файл был модифицирован по ошибке и нам нужно его вернуть
есть подсказка в `git status` 
```

```

#git diff
Команда `git diff` показывает **разницу между файлами**, 
в зависимости от того, **какое состояние ты сравниваешь**. 
По умолчанию она сравнивает **рабочую директорию** с **индексом (staging area)**.
```
git diff --staged
```
он показывает что будет записано в `git log` если сделать `git commit`
_____________________________________
что делать если нам нужно чтобы git игнорировал какие-то файлы либо директории
надо создать файл `.gitignore`
и там можно прописать
```
*.log # если нужно все файлы с расширением
log/ # если нужно директорию
file.txt # если нужно определенный файл
```
